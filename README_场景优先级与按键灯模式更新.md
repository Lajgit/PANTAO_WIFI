- `PanTao_Ctrl_场景灯效与分组覆盖_完整工程.zip`

## 2. 最终优先级

### 2.1 收到按键灯指令

按键灯指令只启用指定分组的覆盖状态：

```text
全局场景先生成全部灯珠颜色
        ↓
已受控按键分组覆盖指定范围
        ↓
统一刷新Light1和Light2
```

结果：

- 指定分组按常亮、常灭或闪烁执行。
- 未受控灯珠继续执行原有全局场景。
- 后续收到新的按键灯指令，可以再次覆盖对应分组。

### 2.2 收到全局场景指令

有效场景指令 `0x00～0x02` 到达控台时：

1. 更新 `Scene`。
2. 清除9个分组全部按键灯覆盖状态。
3. 强制重新生成全局场景，即使场景编号与当前相同也会重新覆盖全部灯珠。

结果：

- 全局场景指令可以覆盖之前所有按键灯指令。
- 之后再收到按键灯指令，对应分组仍可重新覆盖全局场景。

## 3. 通信协议

### 3.1 Unity到球盘：控台按键灯

```text
AA 重传ID 唯一ID 01 17 00 00 位置编号 颜色编号 01 模式 CRC_H CRC_L 55
```

模式：

```text
0x00：常亮
0x01：常灭
0x02：闪烁，亮100ms、灭100ms，完整周期200ms
```

球盘继续转发为：

```text
AA 00 球盘动态ID 02 05 00 00 位置编号 颜色编号 00 模式 CRC_H CRC_L 55
```

球盘代码没有改变，`ExpandCode` 会原样转发，因此可直接传递新的闪烁模式 `0x02`。

### 3.2 Unity到球盘：全局场景

```text
AA 重传ID 唯一ID 01 07 00 00 00 场景编号 01 00 CRC_H CRC_L 55
```

场景：

```text
0x00：所有WS2812九种颜色呼吸亮灭
0x01：所有WS2812白色常亮
0x02：所有WS2812红色常亮
```

球盘继续转发为控台功能码 `0x02/0x03`。

## 4. 位置编号

按照用户本次提供的编号保留：

| 位置 | 业务名称 | 硬件范围 |
|---|---|---|
| `0x00` | 孙悟空升级按键；游戏说明按键 | `Light1[18..25]` |
| `0x01` | 猪八戒升级按键 | `Light1[26..33]` |
| `0x02` | 沙悟净升级按键 | `Light1[34..41]` |
| `0x03` | 孙悟空发射按键 | `Light1[42..49]` |
| `0x04` | 猪八戒发射按键 | `Light1[50..57]` |
| `0x05` | 沙悟净发射按键 | `Light1[58..65]` |
| `0x06` | 右上四联按键组，业务名称未指定 | `Light1[0..17]` |
| `0x07` | 旋钮按键 | `Light2[8..15]` |
| `0x08` | 数码管按键 | `Light2[0..7]` |

注意：用户给出的“孙悟空升级按键”和“游戏说明按键”都为 `0x00`，因此当前代码按同一个灯组处理；`0x06`仍保留原来的右上四联硬件分组，但没有分配新的业务名称。若“游戏说明按键：0x00”是笔误，应将Unity侧改发 `0x06`，或者确认后再改映射。

## 5. 具体修改

### 5.1 `Task/Inc/LightTask.h`

原代码：

```c
#define CTRL_LIGHT_MODE_BLINK 0x03U

bool CtrlLightGroup_Set(uint8_t position, uint8_t color, uint8_t mode);
```

修改后：

```c
#define CTRL_LIGHT_MODE_BLINK 0x02U

bool CtrlLightGroup_Set(uint8_t position, uint8_t color, uint8_t mode);
void CtrlLightGroup_ClearAll(void);
```

原因：协议将闪烁模式由 `0x03` 改为 `0x02`，并增加供场景指令清除覆盖状态的接口。

影响范围：只影响控台功能码 `0x05` 的模式解析和功能码 `0x03` 的场景优先级。

### 5.2 `Task/Src/CommTask.c`，函数 `USART_Deal()`

原代码：

```c
case 0x03:
    if (mesg->Data4 <= SCENE_PLAYING)
        Scene = (Scene_t)mesg->Data4;
    break;
```

修改后：

```c
case 0x03:
    if (mesg->Data4 <= SCENE_PLAYING)
    {
        Scene = (Scene_t)mesg->Data4;
        CtrlLightGroup_ClearAll();
    }
    break;
```

原因：收到全局场景后需要取消此前全部按键灯覆盖。

影响范围：非法场景编号仍被忽略；有效场景 `0x00～0x02` 会清除全部分组覆盖。

### 5.3 `Task/Src/LightTask.c`

新增：

```c
void CtrlLightGroup_ClearAll(void)
{
    for (uint8_t i = 0; i < CTRL_LIGHT_GROUP_NUM; i++)
    {
        CtrlLightGroupState[i].Enabled = 0U;
        CtrlLightGroupState[i].Dirty = 0U;
        CtrlLightGroupState[i].BlinkOn = 0U;
    }

    GlobalSceneForceRefresh = 1U;

    if (Scene == SCENE_LESSLIGHT)
        ResetGlobalBreath();
}
```

原因：关闭全部分组覆盖并强制全局场景重绘；同一个场景编号重复下发也能覆盖此前按键灯状态。

影响范围：不改变全局场景函数、不改变分组灯珠范围、不影响未受控灯珠的正常场景效果。

## 7. 编译和回归

使用上一版球盘工程，无需再次修改球盘源码：

```text
PanTao_2/MDK-ARM/PanTao_2.uvprojx
```

控台使用本次更新工程：

```text
PanTao_Ctrl/MDK-ARM/PanTao_Ctrl.uvprojx
```

在Keil中执行：

```text
Project -> Rebuild all target files
```

回归步骤：

1. 上电确认默认场景0九色呼吸。
2. 发送单个按键灯指令，确认只覆盖指定分组。
3. 确认其他未受控灯珠继续全局呼吸。
4. 重复发送场景0，确认此前所有按键覆盖都被清除。
5. 发送场景1或场景2，确认全部灯统一白色或红色。
6. 再发送按键灯指令，确认对应分组重新覆盖全局场景。
7. 验证模式 `0x03` 已不再作为闪烁，只有 `0x02` 可触发闪烁。
